# SOP: Session Logging

After every significant working session, log it using this format.

## Format

```markdown
### [YYYY-MM-DD HH:MM] - [Short task name]
- **Done:** [Summary of what was completed]
- **Key decisions:** [Technical or design choices agreed on]
- **Open items:** [Unfinished or needs follow-up]
- **Context to remember:** [Important info for the next session]
```

## Where to save

`_memory-logs/YYMM-log.md` - one file per month.

Example: `_memory-logs/2605-log.md` for May 2026.

## When to log

- After completing a major feature or task
- Before ending a session
- When a key decision is made
- Before switching to a completely different task
